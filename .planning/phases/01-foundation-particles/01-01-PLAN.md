<task type="auto">
  <name>Task 3: Create custom particle engine lifecycle hook</name>
  <files>components/particles/useParticleEngine.ts</files>
  <action>
Create `components/particles/useParticleEngine.ts` with a custom React hook that manages tsParticles engine initialization:

1. **Hook signature**: `useParticleEngine()` returns `{ isReady: boolean }`

2. **Implementation**:
   - Use `useState(false)` for `isReady` state
   - Use `useEffect` with empty deps to initialize once on mount
   - Call `initParticlesEngine()` from `@tsparticles/react`
   - Pass `loadSlim` from `@tsparticles/slim` to the engine initializer
   - Set `isReady(true)` after engine loads successfully
   - Handle errors gracefully (catch and log, but don't crash)

3. **Pattern** (from ARCHITECTURE.md Pattern 1):
```typescript
import { useEffect, useState } from 'react';
import { initParticlesEngine } from '@tsparticles/react';
import { loadSlim } from '@tsparticles/slim';

export const useParticleEngine = () => {
  const [isReady, setIsReady] = useState(false);

  useEffect(() => {
    initParticlesEngine(async (engine) => {
      await loadSlim(engine);
    })
      .then(() => setIsReady(true))
      .catch((error) => console.error('Failed to load particle engine:', error));
  }, []);

  return { isReady };
};
```

This hook encapsulates the engine initialization complexity, making the main component cleaner.
