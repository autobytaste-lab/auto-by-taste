<task type="auto">
  <name>Task 1: Define z-index CSS variable hierarchy</name>
  <files>index.html</files>
  <action>
Update the `<style>` section in index.html to add CSS variables for z-index management:

1. **Add CSS variables** in the `:root` selector at the top of the style block:
```css
:root {
  --z-particles: 0;
  --z-background: -10;
  --z-content: 10;
  --z-glass-cards: 15;
  --z-navbar: 20;
  --z-modal: 50;
  --z-fab: 100;
}
```

2. **Enhance glass-card class** with consistent values and browser fallbacks:
```css
.glass-card {
  background: rgba(255, 255, 255, 0.03);
  backdrop-filter: blur(10px);
  -webkit-backdrop-filter: blur(10px); /* Safari fallback */
  border: 1px solid rgba(255, 255, 255, 0.1);
  position: relative;
  z-index: var(--z-glass-cards);
}

/* Fallback for browsers without backdrop-filter support */
@supports not (backdrop-filter: blur(10px)) {
  .glass-card {
    background: rgba(255, 255, 255, 0.08);
  }
}
```

3. **Add glass-card-strong variant** for sections needing more opacity:
```css
.glass-card-strong {
  background: rgba(255, 255, 255, 0.05);
  backdrop-filter: blur(15px);
  -webkit-backdrop-filter: blur(15px);
  border: 1px solid rgba(255, 255, 255, 0.15);
  position: relative;
  z-index: var(--z-glass-cards);
}
```

4. **Keep existing styles** (.text-gradient, @keyframes float, .animate-float, .hologram-glow) unchanged.

These variables establish a clear hierarchy preventing z-index conflicts (PITFALLS.md Pitfall 5). The stronger glass variant provides flexibility for sections needing more prominence.
