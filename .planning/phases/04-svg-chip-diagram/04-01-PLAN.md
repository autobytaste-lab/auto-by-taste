Replace the stub with the full ChipDiagram component implementation. Follow the architecture from 04-RESEARCH.md exactly.

**Component structure:**
1. Filter `chips` array for M4 generation, exclude Ultra: `chips.filter(c => c.generation === 'M4' && c.variant !== 'Ultra')`
2. `useState<'base' | 'Pro' | 'Max'>('base')` for active variant
3. `useRef<HTMLDivElement>(null)` for container, pass to `useInView(containerRef)`
4. `useReducedMotion()` for accessibility
5. Four `useCountUp` calls for cpu, gpu, tops, bandwidth — triggered by `inView`
6. Display values: use final chip values when `reducedMotion` is true, animated values otherwise

**HTML section (above SVG):**
- Container: `<div ref={containerRef} className="w-full max-w-2xl mx-auto px-4">`
- Tab bar: `<div role="tablist" aria-label="M4 chip variant selector">` with 3 `<button role="tab">` elements
- Active tab: `aria-selected="true"`, styled with Tailwind `bg-blue-600 text-white rounded-lg px-4 py-2`
- Inactive tabs: `text-slate-400 hover:text-white px-4 py-2`
- Tab bar styling: `flex gap-2 mb-4 justify-center`

**SVG section:**
- `<svg viewBox="0 0 400 300" width="100%" preserveAspectRatio="xMidYMid meet" role="img" aria-label="{variant name} chip architecture diagram">`
- Use `<defs>` for reusable rounded-rect shapes to save element count
- 4 block regions using `<use>` referencing defs:
  - CPU block: x=10, y=10, w=185, h=120 — blue fill (e.g., fill="#1e3a5f" fillOpacity=0.3), label "{cpu}-core CPU ({P}P+{E}E)", className animate-core-glow when shouldAnimate
  - GPU block: x=205, y=10, w=185, h=120 — purple fill (e.g., fill="#2d1f5e" fillOpacity=0.3), label "{gpu}-core GPU", className animate-core-glow when shouldAnimate
  - Neural Engine: x=10, y=145, w=185, h=90 — emerald fill (e.g., fill="#064e3b" fillOpacity=0.3), label "Neural Engine", "{tops} TOPS"
  - Unified Memory: x=205, y=145, w=185, h=90 — cyan fill (e.g., fill="#164e63" fillOpacity=0.3), label "Unified Memory", "{maxMemory}GB", className animate-memory-shimmer when shouldAnimate
- Spec bar: x=10, y=250, w=380, h=40 — summary line with memory bandwidth "{bw} GB/s bandwidth"
- Text styling: fill="white" for labels, fontSize 14-16 in viewBox units for readability at 320px, textAnchor="middle" centered in blocks

**Key details:**
- Accept optional `className?: string` prop for Phase 5 container styling
- Do NOT re-animate on variant switch (per research recommendation). After first inView trigger, subsequent variant changes show final values instantly
- Use `key={activeVariant}` on the SVG element is NOT recommended (would cause remount flicker). Instead accept instant number updates on tab switch after initial animation
- Stroke for block borders: stroke="currentColor" strokeOpacity=0.2
